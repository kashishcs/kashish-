Algorithm Visual Playground — Case Study

Challenge

Teaching algorithmic intuition is difficult because students often see code but not the state transitions. The challenge was to create a responsive, accurate visualization engine that can animate algorithms step-by-step while remaining performant for large inputs.

Solution

Built a Canvas-based engine with an event-driven state machine. Algorithms are instrumented to emit state snapshots; the renderer consumes snapshots and animates transitions using requestAnimationFrame. To maintain performance, I implemented incremental diffing and batched DOM updates.

Technical Details

  • Renderer: Canvas + layered compositing for nodes, edges, and UI overlays.
  • State management: immutable snapshots with efficient diffing to compute minimal redraws.
  • Algorithms: visual implementations of QuickSort, MergeSort, BFS, DFS, and recursive tree traversals with step controls and breakpoints.

Learning

I learned how to balance correctness and performance, how to design an API for instrumenting algorithms, and how to present complexity trade-offs visually.

Live Demo

Tech stack: JavaScript (Canvas), Python for backend data generation, Webpack, Jest tests for algorithm correctness.

Plagiarism Checker — Case Study

Challenge

Detecting similarity across large codebases and long text documents requires algorithms that are both accurate and efficient. The system must handle obfuscation, variable renaming, and formatting differences.

Solution

Implemented a hybrid approach combining tokenization, normalized hashing (rolling hash / Rabin-Karp), and semantic fingerprinting using n-grams. For code, AST-based normalization reduces false positives due to formatting. For text, TF-IDF and cosine similarity are used for ranking.

Technical Details

  • Preprocessing: lexical normalization, AST parsing for code, stopword removal for text.
  • Algorithms: rolling hash for substring detection; MinHash + LSH for scalable similarity search.
  • Performance: chunked processing, streaming I/O, and parallelization using multiprocessing.

Learning

I learned about trade-offs between precision and recall, the importance of normalization, and how to scale similarity detection to larger datasets.

Mini Demo

Tech stack: Python (NLTK, scikit-learn), AST parsing, MinHash, Flask for API.

Student Result Analyzer — Case Study

Challenge

Presenting student performance data in a way that surfaces actionable insights without overwhelming the user. The system must handle missing data, noisy inputs, and provide clear recommendations.

Solution

Built a pipeline that ingests CSV data, cleans and normalizes scores, computes derived metrics (moving averages, variance, subject correlations), and renders interactive charts with Chart.js. The UI includes filters, drill-downs, and exportable reports.

Technical Details

  • Data pipeline: Python scripts for ETL, Pandas for cleaning, and Node.js for serving processed JSON.
  • Visualization: Chart.js with custom plugins for annotations and trendlines.
  • UX: progressive disclosure of insights and contextual recommendations.

Learning

I learned how to design metrics that are interpretable, how to handle missing data responsibly, and how to communicate uncertainty to users.

Interactive Chart

Tech stack: Chart.js, Pandas, Node.js, REST API, unit tests for analytics functions.
